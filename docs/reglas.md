# Reglas del Proyecto Ainglo

## 1. Consulta obligatoria de documentación

Antes de crear, modificar o proponer cualquier funcionalidad, **siempre** debes revisar los archivos:

- `README.md`: para entender la visión, objetivos y contexto general del proyecto.
- `Estructura.md`: para conocer la arquitectura, organización de carpetas/archivos y convenciones de nombres.

Esto asegura que todo el desarrollo siga la estructura y filosofía definida, evitando trabajo duplicado o fuera de estándar.
